# SEO/LLMO 監査

戦略監査 + 技術監査を統合し、「次に何を作るべきか」を判断する。

---

## 手順

### Phase 1: 現状把握
1. サイトの全ページ構成を確認（`src/app/` 配下）
2. 既存コンテンツのインベントリ作成
3. `public/ai.txt` と `public/llms.txt` の最新性を確認

### Phase 2: 戦略監査（4つの観点）

| 観点 | 分析対象 |
|------|---------|
| LLMO/AI引用 | ai.txt/llms.txt のカバレッジ、構造化データの網羅性、AIが引用しやすい構造か |
| SEO権威性 | メタデータ設定、キーワード対応状況、コンテンツギャップ |
| ユーザーコンテンツ | 求職者・若者向けコンテンツの充足度 |
| 3トラック連動 | SEO/LLMO/User トラック間の内部リンク、エンティティ一貫性 |

### Phase 3: 技術監査（5つの領域）

| 領域 | 監査内容 |
|------|---------|
| メタデータ | 全ページの title/description/OG 設定状況 |
| 構造化データ | Schema.org の実装状況、@id チェーンの整合性 |
| LLMO対策 | robots.txt、ai.txt、llms.txt、セマンティックHTML |
| パフォーマンス | Image最適化、RSC/Client比率、フォント |
| セキュリティ | ヘッダー、エラーページ、API安全性 |

### Phase 4: 統合レポート
- 総合スコア（10点満点）
- 優先度付き改善アクションリスト
- **「次に書くべき記事 TOP 5」を提案**

- 全提案にデータの根拠を付ける
